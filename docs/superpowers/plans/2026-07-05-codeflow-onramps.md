---
name: triage
description: 把外来 issue（及声明开启时的外部 PR）推过 triage 状态机——归类、验证、必要时拷问、写 agent-ready 简报；被拒的 enhancement 沉淀进 out-of-scope 知识库。仅当用户主动用 `/codeflow:triage` 指令调用时使用 —— 不要在普通对话里自行触发。

正文骨架（设计文档 §2 逐项展开，追踪器操作一律指 issue-tracker.md）：
1. **角色与状态机**：2 类别 × 5 状态、恰好一类别一状态不变量、转移规则（未标→needs-triage；needs-info 收到回复回 needs-triage；维护者可随时覆盖，异常转移先问）。PR 面（仅 GitHub+声明开启）：PR 是带代码的 issue，同角色同状态同机器；ready-for-agent 对 PR=brief 已附、agent 对既有 diff 做下一步。
2. **AI 声明**：发到追踪器的每条评论开头必须是 `> *This was generated by AI during triage.*`（英文原句，面向公共受众；正文中文解释此规则）。
3. **调用方式**：维护者自然语言描述意图，四类示例（看看有什么要办 / 看 #42 / 把 #42 挪到 X / 有什么 agent 能领的）。
4. **呈现待办**：三桶按最旧优先（未标 / needs-triage / needs-info 且报告人有新动静）；PR 面开启时并入外部 PR 并标 `[PR]`/`[issue]`（discovery 只捞外部 PR，点名的 PR 一律办）；给计数与一行摘要，维护者挑。
5. **办一个 issue/PR 五步**：gather（全文+旧笔记不重问；冗余检查按领域概念搜既有实现并报告搜过哪里；先例检查读 `docs/jljskills/codeflow/out-of-scope/` 全部文件按概念相似匹配）→ recommend（类别+状态+理由+代码库摘要，等指示）→ verify（bug 按步骤复现；PR checkout 跑相关测试验 diff；三态结果：确认/失败/细节不足=强 needs-info 信号）→ grill（需要补细节时指路 `../grill-with-docs/SKILL.md` 逐题拷问、术语当场落笔）→ apply（五状态动作；wontfix 三分支：已实现→指路所在处不写知识库 / bug 拒绝→礼貌解释关闭 / enhancement 拒绝→写 out-of-scope 并链接后关闭）。
6. **快速覆盖与续接**：点名挪状态直达（确认动作后执行、跳过拷问，挪 ready-for-agent 时问要不要 brief）；有旧笔记先读、报告人答了的不重问。
7. **needs-info 笔记模板**（「已确认的 / 还需要你（@报告人）提供的」两段，问题必须具体可答）。
8. 「to-issues 产出的 issue 已 agent-ready，不要 triage」的边界一句。
9. brief 写法与知识库规则分别指 references/agent-brief.md、references/out-of-scope.md。

- **Phase 2 复现+最小化**：跑红确认三点（是用户描述的那个失败而非隔壁失败、可重复、症状已捕获）；最小化逐项裁、每裁一刀重跑、剩下每项都承重才算完；不复现不最小化不前进。
- **Phase 3 假设**：一次 3–5 个成列、可证伪（格式「若 X 是因，则改 Y 消失/改 Z 更糟」）、说不出预测就是玄学弃掉或磨尖；列表先给用户看（常见秒级重排）但用户不在场不阻塞。
- **Phase 4 插桩**：每个探针对应一个预测、一次一变量；调试器/REPL 优先于日志、绝不「全打出来再 grep」；日志带唯一前缀如 `[DEBUG-a4f2]` 方便一把清；性能分支：先基线测量（计时/剖析/查询计划）再二分，先测量后修。
- **Phase 5 修复+回归测试**：修复前先写回归测试——但仅在**正确 seam** 存在时（测试在调用现场按真实 bug 模式行使；seam 判断 Read `../design-rules/SKILL.md`）；无正确 seam 本身就是发现——记录、标给下一阶段；有则：最小复现变失败测试→看它红→修→看它绿→回原始未最小化场景重跑 Phase 1 回路。
- **Phase 6 清理+复盘**：五条 checkbox（原始复现已不复现 / 回归测试过或无 seam 已记录 / `[DEBUG-...]` 全清（grep 前缀）/ 一次性 harness 删净或移到显式 debug 位置 / 验证过的假设写进 commit 信息）；复盘问「什么本可防住这个 bug」——答案是架构性的（无好 seam、调用方纠缠、隐藏耦合）就带着具体细节指路 `/codeflow:improve-arch`，且在修完之后再提不在之前。
