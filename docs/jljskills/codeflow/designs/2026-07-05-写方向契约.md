# 写方向契约落盘 — 设计树结论与 interface 草图

## 分支结论

- **约束**：README 随插件分发（engineering 安装缓存实证），但不被运行时自动装载 → 运行时红线必须留在各 SKILL.md；跨插件仅能按名互指；契约文本不得含时序引导（用户已否决）；契约天然分两半边（写回侧 / 只读侧）。
- **seam 位置**（用户裁定）：README 各半边 + ADR-0003 记判据。what 在 README（插件使用者可见），why 在 ADR（防未来审查重提），运行卡点在 SKILL.md 红线——三层分工不重叠、不复述。
- **interface 形状**：两 README 各设同名小节「与对侧的分工（写方向契约）」，按名互指；figma-optimize 三个评审 SKILL.md 的「独立插件」整段收窄为一句运行红线 + `../../README.md` 指针（依 CONTEXT.md「装载式指针」：消费点不留摘要）；figma2web 的 8 个 skill 不加新段（避免蔓延，其红线已隐含只读）。
- **藏什么**：契约正文（写方向判据、读到即确认、无管线关系）全部吸进 README 单点；会话记忆降级为指针。
- **测试存活**（登记面自检）：根 README 目录树补 figma-optimize README 行、figma2web README 备注去掉已死的「旁路」；grep 验证「独立插件」逐字复述面 3→0。

## interface 草图（契约的载体结构）

- `plugins/figma-optimize/README.md`「与 figma2web 的分工（写方向契约）」——写回侧半边权威源：唯一写回 Figma、绝不落项目代码、报告仅旁产物。
- `plugins/figma2web/README.md`「与 figma-optimize 的分工（写方向契约）」——只读侧半边权威源：只读 Figma 绝不写回、读到即确认（不设质量 gate、变更由人经 re-\* 重新声明）。
- `docs/jljskills/codeflow/adr/0003-*.md`——判据与三个否决项。
- **不变量**：两半边按名互指；任何 skill 文件不复述契约正文，只留单句运行红线 + 指针；全仓文档不得出现两插件间的时序建议。
- **错误模式（漂移防线）**：新增 figma-optimize skill 忘守契约 → README 契约节是评审查验点；未来审查提反向方案（如 component 搬回）→ ADR-0003 挡回。
- **依赖策略**：in-process（纯文档结构），无 adapter。
