# 双插件分工按写方向裁，figma2web 读到即确认

figma2web 与 figma-optimize 的分工判据不是「产出 vs 优化」，而是**写方向**：figma-optimize 是唯一有权写回 Figma 的一侧、绝不落项目代码；figma2web 只读 Figma、只写项目文件。配套契约：figma2web **读到即确认**——用户发起管线即声明「产出当前这张图」，设计稿是读取时刻的权威快照，管线不设设计质量 gate、不提示「先去优化」，设计稿变更由人经 re-\*（独立意图入口）重新声明。两插件之间**只有人的编排，没有管线关系**。2026-07-05 依此判据把 component 从 figma2web（扫代码沉淀）裁给 figma-optimize（散件收编）。契约正文落两个 plugin README 各自半边、按名互指；本 ADR 只记判据与否决，防未来审查重提。

## Considered Options

- **按「优化 vs 产出」分工**（被否决）：遇骑墙技能裁不动——config 读 Figma 规范板，按「做的是设计图的工作」也能归 figma-optimize；写方向一裁即明。
- **在 README 写推荐时序（先优化后转码）**（被否决）：暗示 figma2web 对输入设计有质量预期，违背「读到即确认」；技能组是辅助人的工具，不是流程管制。
- **figma2web 增设计质量 gate / 优化提示**（被否决）：输入契约是无条件信任用户确认的快照，跨插件只做「这事不归我管，在那边」的边界指路。
