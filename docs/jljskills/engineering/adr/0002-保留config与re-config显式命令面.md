# 保留 config/re-config、page2doc/re-page2doc 显式命令面

2026-07-04 架构审查提议把 re-\* 折叠回 base skill（reconciler 语义：首建 = 空基线 diff 的特例，命令面 4→2）。否决：这些是干活型 skill，**命令面的每个入口是一次意图声明**——「从零建标准」与「Figma 改了、只同步差异」是两个用户意图，各自编码了前置预期与产出范围；并成一个命令后，interface 上省掉的入口会变成调用者脑子里多出来的推断规则（「它会怎么判断我的处境」）。语义密度在命令名上，不在入口数量上。

## Considered Options

- **reconciler 折叠**（被否决）：算法上首建确是增量的退化路径（全项 NEW），implementation 视角成立；但 interface 为调用者服务，不为算法服务。未来审查的删除测试仍会推出此案——以本 ADR 为准，不重开。
- 机制层的复述漂移风险另行处理：re-\* 对 base 流程的压缩复述改为显式指针（不折叠命令面也能收敛漂移面）。
