3. Token → User row: two-tier dependency

Looks up the row by firebase_uid, enforces no row → 404, inactive → 403, email mismatch → 403.

⚑ War story — async wraps a sync helper

A sync yield-dep around user_id_scope blew up with "Token was created in a different Context". FastAPI runs __enter__/__exit__ in two different copy_context() threadpool calls → the contextvar set() couldn't be reset(). Async wrapper keeps set + reset in one Context.

Recap — cradle to grave

  • Firebase authenticates; the backend mirrors. One row per firebase_uid; reads 404 rather than auto-create.
  • Deactivation is soft + delayed. is_active + 7-day window; Firebase identity dies immediately.
  • Messy realities handled explicitly. Idempotent deletes · non-fatal cleanup · async/sync contextvar split.
