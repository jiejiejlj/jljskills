# Issue 模板

四节顺序固定，逐节写：

## Parent

源头 issue 的引用（编号/URL）。源材料是从既有 issue 拆解出来的才写这节；直接从 PRD 或计划拆分时，整节省略。

## 要构建什么

描述这一切片建成后端到端的行为，不按 schema/API/UI 逐层罗列实现步骤。

文件路径与代码片段的取舍规则见 `../../to-prd/SKILL.md` 第 3 节，此处不重复。

## 验收标准

- [ ] 标准一
- [ ] 标准二
- [ ] 标准三

checkbox 清单，每条独立可验证；标准描述"做完是什么样"，不是"要做哪些步骤"。

## Blocked by

- 真实的阻塞 issue 标识（编号/URL）

无阻塞时写「无——可立即开工」，不要留空。
