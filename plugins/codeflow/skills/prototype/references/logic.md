# 逻辑原型

一个可交互终端小程序，让用户徒手驱动一个状态模型。问题关于**业务逻辑、状态迁移或数据形状**时用这个分支——那种纸面上看着合理，只有真推几个用例才会觉得不对的东西。

## 什么时候该走这条分支

- 「我不确定这个状态机能不能处理先 X 后 Y 的边界情况」
- 「这个数据模型真能表示……这种情况吗」
- 「我想在写 API 之前先摸一下它该是什么样子」
- 任何用户想**按按钮、看状态变**的场合。

问题是「这东西该长什么样」——分支选错了，回 [ui.md](ui.md)。

## 做法

### 1. 把问题写下来

写代码前，先写清楚要验证的状态模型和问题，一段话，放进原型的 README 或文件顶部注释。答错问题的逻辑原型是纯浪费——把问题写明白，不管用户此刻在不在场，之后都能核对它有没有真的被回答。

### 2. 选语言

用宿主项目本来在用的语言。项目没有明显的运行时（比如纯文档仓库），先问用户。

工具链跟着项目既有约定走——不要为了原型另装一套包管理器或运行时。

### 3. 把逻辑关进一个可移植的模块

把真正回答问题的那部分逻辑，关进一个小、纯的 interface 后面，让它以后能原样搬进正式代码库。围着它的 TUI 是要扔的，逻辑模块不是。

具体形状看问题：

- **纯 reducer**——`(state, action) => state`。适合动作是离散事件、状态是单一值的场合。
- **状态机**——显式的状态与迁移。适合「当前哪些动作合法」本身就是问题一部分的场合。
- **一组纯函数**，作用在一个普通数据类型上。适合没有隐含「当前状态」、只有变换的场合。
- **有清晰方法面的类/模块**，当逻辑确实拥有持续存在的内部状态时用。

按问题选形状，不是按哪个接 TUI 最省事。保持纯——不掺 I/O、不掺终端代码、不用 `console.log` 做控制流。TUI 引用它、调用它；反过来不行。

这才是原型能活过自己生命周期的原因：问题答完，验证过的 reducer / 状态机 / 函数组能直接搬进正式模块，TUI 外壳整个删掉。

### 4. 搭一个刚好能显出状态的最小 TUI

搭成一个**轻量 TUI**——每个 tick 清屏（`console.clear()` / `print("\033[2J\033[H")` 或等价物），重新渲染整帧。用户始终看到一个稳定的画面，不是越滚越长的历史输出。

每一帧分两部分，顺序固定：

1. **当前状态**，格式化到方便肉眼对比（一行一个字段，或格式化 JSON）。字段名/小节标题用**粗体**，次要信息（时间戳、ID、派生值）用**暗色**。原生 ANSI 转义码就够——`\x1b[1m` 粗体、`\x1b[2m` 暗色、`\x1b[0m` 复位；项目里没现成的样式库就别为这个引入一个。
2. **键位提示**，列在底部：`[a] 新增用户  [d] 删除用户  [t] 时钟走一格  [q] 退出`。键位加粗、说明调暗，或反过来，怎么读着顺就怎么来。

行为：

1. **初始化状态**——一个内存里的对象/结构体。启动时渲染第一帧。
2. **读一次按键（或一行输入）**，分发给对应的处理函数去改状态。
3. **每次动作后重渲染**整帧——不是追加，是替换。
4. **循环直到退出。**

整帧应该一屏能看全。

### 5. 让它一条命令能跑

把脚本挂到项目既有任务运行器上（`package.json` scripts / `Makefile` / `justfile` / `pyproject.toml`）。用户跑 `pnpm run <原型名>` 或等价命令就行，不用记路径。

宿主项目没有任务运行器，就把命令写在原型 README 顶部。

### 6. 交给用户

把运行命令给用户，让他们自己驱动。有意思的时刻是他们说「等等，这不该发生」或「奇怪，我以为 X 应该不一样」——那些是**想法本身**的 bug，这才是原型的全部意义。他们想加新动作就加，原型是会演化的。

### 7. 把答案记下来

原型完成任务后，唯一值得留的是答案。用户在场，当场问他学到了什么；不在场，在原型旁留一份 `NOTES.md`，等答案回来（或你看过会话记录后自己）填上，再删原型。

## 反模式

- **不加测试。** 需要测试的原型已经不是原型了。
- **不接正式数据库。** 除非问题本身就是关于持久化，否则用内存存储。
- **不做泛化。** 不搞「以后要是想支持 X」。原型只答一个问题。
- **不让逻辑和 TUI 糊在一起。** reducer / 状态机里出现 `console.log`、prompt 或终端转义码，就不再可移植了。TUI 只该是纯模块外面薄薄一层壳。
- **不把 TUI 外壳送进生产。** 外壳是为了手动在终端里驱动而优化的，值得留下的是它背后的逻辑模块。
