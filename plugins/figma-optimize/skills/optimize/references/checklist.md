# 内置评审清单

本插件的**核心知识资产**:一份以「利于开发落地成代码」为导向的独立设计评审清单。**结构类 (A) 与切图友好类 (B) 优先级最高 (P0)**——它们最直接决定开发能否顺畅实现。

评审时把本清单与**运行时抽到的项目标准**(色板 / 字阶 / 网格基数,见 flow.md P1)合起来用:清单给「查什么」,项目标准给「对不对得上」。抽不到项目标准时,退回本清单的通用默认(见文末阈值)。

每条建议呈交时含四要素:**问题 / 改法 / 依据(清单编号) / 严重度(高 / 中 / 低)**。

## snap 网格例外(不误报)
以下合理非网格 / 非 token 值**不报**为问题:

- **hug 宽度**:随内容自适应,天然非整值。
- **1px 描边 / 分隔线**:细节值,不必落网格。
- **组件实例内部**:值由主组件决定,应改主组件而非实例。
- **矢量锚点 / 图标细节**:路径坐标不参与网格审计。

## 判据方式:数值类 vs 视觉 / 结构类
- **数值类**(C / D / E / F):靠 `use_figma` 读 x/y/w/h/fills/字号 等精确值审计,可判定「对不对得上项目标准」。
- **视觉 / 结构类**(A / B / G / H):靠 `get_screenshot` + `get_design_context` 判断,依赖对布局意图的理解。

## 通用默认阈值(抽不到项目标准时退回)
- **间距网格基数**:8pt(4pt 为可接受半档);优先以运行时抽到的基数为准。
- **近似值容差**:同类值差 ≤2px 视为「应统一」的近似不一致。
- **色板归属**:以运行时 variables/styles 为准;无变量则以代表帧里高频色归纳的色板为准,缺口对话补。
