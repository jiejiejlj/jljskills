# component 详细流程

交付前把界面稿里的重复散件收编成 Figma components, 产一份收编报告.

## 核心心法

面向**设计师**, 在界面稿交付 / 长期维护前, 把稿内**重复出现却仍是散件**的元素**收编**成 Figma components, 让设计稿从「拷贝粘贴维护」升级为「组件单点维护」, 开发拿到的也是成体系的组件而非散件. AI 在这里充当**同构检测器 / 收编执行者**: 只揪可验证的同构重复, 该不该成组件, 叫什么名, 变体怎么归并由设计师拍板.

**收编 = 外观零变化.** 成立组件, 原位替换为实例后, 每一处视觉必须与收编前一致 — 组件化改的是结构, 不是样子.

差异超出 variants / 实例覆写能表达的候选**不硬并**, 把差异摆给用户裁定拆分.

> 通用纪律见 SKILL.md 红线, 本文件不复述.

## P0 — 前置校验

1. figma-mcp 可用(未认证 → 先跑认证流程).
2. 向用户索取**待收编的界面稿链接 / 范围**(单页 / 若干区块). **不落盘, 每次现问.**

## P1 — 装载判据 + 摸底

1. **Read [`../../figma-facts.md`](../../figma-facts.md) 装载共享判据**(硬性步骤): 判据正文以它为唯一权威, 不在本文件复述.
2. `get_metadata` 取全页结构, 按节点类型分三类:
   - **已是实例**(`INSTANCE`)— 不动, 只在报告里计数;
   - **已有组件**(`COMPONENT` / `COMPONENT_SET`)— 作为游离克隆比对的基准;
   - **其余散件** — 进入 P2 扫描.

## P2 — 扫描候选(同构判据)

对散件做**结构签名**比对(递归的 节点类型 + 图层结构 + auto-layout 骨架; 忽略文本内容与画布位置, 样式差异记为潜在变体轴), 产三类候选:

- **重复散件**: 结构签名相同的散件 ≥2 处.
- **游离克隆**: 与某已有 component 结构签名相同, 却不是它的实例.
- **单处自成一体**: 仅 1 处, 但按设计系统惯例显然应组件化的交互 / 复用单元(按钮 / 输入框 / 卡片 / 标签等).

`get_screenshot` 逐候选取证; 各处差异(文本 / 色 / 状态)逐条记录, 预判归入「变体轴」「实例覆写」还是「超出表达, 建议拆分」.

## P3 — 呈交候选清单(HARD GATE)

每候选呈交: 出现处清单, 建议组件名, 变体归并方案(哪几处并为哪些 variants), 各处差异点与表达方式. 用户逐个:

- **采纳** — 进入 P4 收编.
- **跳过** — 记入报告(跳过).
- **调整** — 改名 / 改归并方案 / 拆分, 修正后重新呈交.

## P4 — 收编写回(仅采纳项)

1. 按 [cookbook.md](cookbook.md) 配方执行: 成立 component(多状态先归并变体)→ 其余出现处**原位替换为实例**(保层级序, 保几何, 差异走变体切换 / 实例覆写).
2. **每处替换完 `get_screenshot` 与收编前截图比对** — 外观零变化; 变了即修正或回滚该处, 再继续下一处.
3. **游离克隆**同法: 替换为对应已有 component 的实例.

## P5 — 复审

- 采纳候选全部收编完毕, 每处均通过零变化比对;
- 或仅剩用户**有意保留**, 并在报告标注的例外.

未达标项回 P3 与用户确认是保留还是继续.

## P6 — 收编报告(可选落盘)

报告含: 成立的组件(名 / 变体 / 收编处数), 游离克隆归位数, 跳过项及原因, 保留例外, 稿内组件化程度变化(实例数 / 散件数, 收编前后).

## 完成标志

采纳候选全部收编且逐处外观零变化; 报告产出.

## 边界

- 只收编既有散件, **不无中生有设计新组件 / 新视觉**.
- 差异超出 variants / 覆写表达力的**不硬并**.
- 命名与「该不该成组件」的最终决定权归设计师; AI 只提供可验证的同构证据.
- 越出「扫描 + 经确认收编 + 出报告」范围即停.

## 审计面(rev 1)

本阶段的 `surface()` 递归子树结构(节点类型 + 图层结构 + auto-layout 骨架, 忽略文本内容与画布位置), 其输出**即同构键** — 同指纹的散件 = 收编候选. 这与 [`../../fingerprint.md`](../../fingerprint.md) 的去重是同一基元, 用于给克隆分组: **component 拿指纹当同构键**(区别于 page 阶段的审计面, 避免混淆).
