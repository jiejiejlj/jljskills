# page 详细流程

交付前给 Figma **界面稿**做「开发者思维」评审并帮改到位, 产一份交付就绪报告.

## 核心心法

面向**设计师**, 在把设计稿交付给开发之前, 以「开发能不能顺畅实现」为尺度审查 Figma **界面稿**并帮改到位, 弥合设计师 ↔ 开发者的交付鸿沟. AI 在这里充当**规则审计员 / 守门人**: 补上设计师缺的那套开发视角, 揪出可验证的规则, 一致性, 可实现性问题, 而不是审查设计规范板本身(那是 standard 阶段).

**AI 只揪可验证的问题, 审美决策权归设计师.**

> 通用纪律见 SKILL.md 红线, 本文件不复述.

## P0 — 前置校验

1. figma-mcp 可用(未认证 → 先跑认证流程).
2. 向用户索取**待审的 Figma 链接 / 范围**(单页 / 若干区块). **不落盘, 每次现问.**
3. 说明本次评审依据: 内置清单(见 [checklist.md](checklist.md))+ 现场从设计规范板抽取的项目标准.

## P1 — 三源级联装载标准(零 config)

1. **Read [`../../figma-facts.md`](../../figma-facts.md) 装载共享判据**(硬性步骤): 库变量误判坑, 三源 API, 写回通用坑, 纪律四条, P1–P5 全程适用, 判据正文以它为唯一权威, 不在本文件复述.
2. **先问 / 找权威标准**: 向用户确认是否存在「设计规范板 / 页」(命名规范 / 变量 / 字阶等体系板), 优先以它为**权威标准来源**.
3. **`use_figma` 读全量**: 对规范板执行三源 API(见 figma-facts), 拿到全量**色板 / 字阶 / 间距**等 token 体系, 作为本次评审的参照标准. 这是主渠道, 数据最全最准.
4. **`get_variable_defs` 仅快览 / 兜底**: 仅用它做快速浏览或第 3 步失败时的兜底, **不作为主渠道**; 是否已 token 化以 figma-facts 的库变量判据为准.
5. **抽不到** → 退回 checklist.md 的通用默认阈值, 并就关键缺口(至少色板 + 网格基数)**对话补齐**.
6. **不生成, 不读写任何 config 文件**; 项目标准仅用于本次评审, 不落盘.

## P2 — 逐区块提取 + 评审

对每个区块:

1. `get_metadata` + `get_screenshot` + `get_design_context` 取结构与视觉.
2. **数值类**(色 / 间距 / 圆角 / 字号)靠 `use_figma` 读 x/y/w/h/fills 精确审计, 对照 P1 装载的项目标准.
3. **字体检测**: 逐文本节点 `getStyledTextSegments(['fontName', 'fontSize'])` 取实际字体与字号, 配合 `node.hasMissingFont` 检测**缺失字体**; 字号 / 字体族与标准字阶比对, 揪出游离字号与非标字体. 配方细节见 [cookbook.md](cookbook.md).
4. **视觉 / 结构类**靠截图 + design_context 判断.
5. 逐维度对照 checklist.md(A–H), 产**带严重度的建议列表**, 每条含: 问题 / 改法 / 依据(清单编号) / 严重度.
6. 命中 snap 网格例外的合理值**不报**: 对 hug 宽度, 1px 描边, 组件实例, 矢量锚点等合理非网格值**不误报**(判据见 checklist.md).

## P3 — 逐条裁定

逐区块, 逐条呈交建议, 用户对每条选:

- **采纳** — 进入 P4 二次选择.
- **跳过** — 记入报告(跳过).
- **再调** — 用户给反馈, AI 修正该条后重新呈交.

## P4 — 采纳项二次选择

每条采纳的问题, 用户再选处置方式:

- **让 AI 改** — 进入 P5 写回.
- **我自改** — AI 给出明确, 可照做的改法说明(改哪个节点, 从什么值改到什么值, 为什么), 交设计师在 Figma 自行修改.

## P5 — 写回 Figma(仅「让 AI 改」项)

1. `use_figma` 写入改动. 配方细节见 [cookbook.md](cookbook.md).
2. **字体归正**(缺失 / 非标字体的可行归正): 归正**只 load 目标字体**, **绝不 load 缺失字体** — 若尝试 load 缺失字体会抛错, 导致整个脚本**原子回滚**; 中文文本经目标字体自动回退渲染, 属预期行为, 不视为问题; 仅当归正方案是「**改成云端不可用的字体**」时, 才降级为「建议」, 不强制写回, 报告标注「受限」.
3. 其余字体类改动云端受限无法执行时 → 同样降级为「建议」, 报告标注「受限」.

## P6 — 复审

复审**数值类**审计项是否达标:

- 理想为审计列表**归零**;
- 或仅剩用户**有意保留**, 并在报告标注的例外.

未达标项回 P3 与用户确认是保留还是继续改.

## P7 — 交付就绪报告

按 [report-template.md](report-template.md) 出报告: 发现的问题, 处置结果(已改 / 跳过 / 待改 / 受限建议), 保留的例外, 给开发的交接 TL;DR; 如本次评审跨到关联的设计规范板, 补填「附: 关联标准板复查」章节.

## 完成标志

- 数值类审计项复审后归零, 或仅剩报告标注的有意例外.
- 结构类, 切图友好类高严重度问题均已处置(改到位或明确跳过并记录).
- 报告产出, 设计师可据它交付开发.

## 边界

- 只审既有稿并按用户裁定优化, **不无中生有设计新界面**.
- **不做审美裁决**, 审美方向归设计师; AI 只揪可验证问题.
- 数值类靠 `use_figma` 精确审计; 对 hug 宽度, 1px 描边, 组件实例, 矢量锚点等合理非网格值**不误报**.
- 越出「评审 + 经确认写回 Figma + 出报告」范围即停.

## 审计面(rev 1)

本阶段喂进 [fingerprint.md](../../fingerprint.md) 的 `surface()` prop 清单(机制正文见该文件, 这里只定义本阶段取哪些 prop):

- **几何**(父相对 x/y/w/h).
- **fills**(含 `boundVariables.color.id`).
- **strokes**.
- **字体**(family / style / size / lineHeight + `boundVariables.fontSize`).
- **autolayout**(gap / padding).

递归子树 → footer 克隆撞同一指纹.
